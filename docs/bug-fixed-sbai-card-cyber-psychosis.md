# Bug Fixed: SBAI 指数卡片情绪表达失真

日期：2026-04-14

## 问题

报告页里的 SBAI 指数卡片虽然信息完整，但视觉气质和产品语义是脱节的：

1. 原卡片是浅色、平静、偏数据看板的样式。
2. 它没有传达出“人快被 AI 写的代码逼疯”的情绪张力。
3. 指标虽然叫 SBAI 指数，但看起来更像普通统计卡，不像情绪已经濒临失控的状态面板。

## 根因

这块 UI 继承的是整页前一版较轻、较亮、较友好的视觉语言：

1. 使用了白底和低对比度青色数字。
2. 缺少故障、压迫、噪声、警告这些更贴近“赛博精神病”氛围的视觉元素。
3. 只有数值和说明文字，没有给出情绪状态层和更强的叙事线索。

## 修复

本次围绕 SBAI 卡片做了定向重设计：

1. 背景改为深色高对比方案，叠加红色危险热点和冷色故障光。
2. 增加扫描线、扫光和轻微抖动动画，让卡片有持续失稳感。
3. 数值增加红/青双重故障残影，强化“系统正在崩”的视觉效果。
4. 新增状态标签，根据 SBAI 数值显示不同情绪等级：
   - 理智抖动
   - 情绪失稳
   - 接近暴走
   - 彻底破防
5. 新增情绪化文案，让数值不再只是冷数据，而是直接对应“被 AI 代码逼疯”的主观体验。
6. 底部补充三段短标签，进一步把叙事收束到“AI 乱写 / 人类破防 / 理智掉线”。

## 验证

建议执行：

```bash
cargo test
```

生成报告页后确认：

1. SBAI 卡片从浅色数据卡变成深色、危险、带故障感的状态卡。
2. 大数字有明显的红青错位残影，不再是单层静态数字。
3. 卡片顶部能显示情绪状态标签，且会随 SBAI 区间变化。
4. 移动端下文字不会溢出，底部标签会自动改为单列排列。
