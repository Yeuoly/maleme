# Bug Fixed: 报告页整体风格割裂，未体现 AI 时代的压迫感

日期：2026-04-14

## 问题

报告页此前虽然已经开始强化 SBAI 卡片，但整体仍然存在明显割裂：

1. 右上角卡片已经偏激烈，但其余区域还是普通数据看板气质。
2. 整页没有形成统一的未来压迫感，视觉语言不够集中。
3. 字体仍然偏系统默认，缺少足够强的科技感和传播辨识度。
4. SBAI 指数虽然已经参与文案分级，但还没有真正驱动全局危险程度。

## 根因

旧实现的问题主要有三点：

1. 主题色只局部作用于 SBAI 卡片，没有向背景、图表、词云和信息面板扩散。
2. 页面主体仍沿用偏浅、偏温和的配色，因此情绪张力被稀释。
3. 标题、数字、标签、正文没有形成分层明确的字体系统。

## 修复

本次做了整页级别的主题重构：

1. 新增基于 `sbai` 的全局主题函数，让整页颜色随危险等级变化。
2. 视觉方向改为：
   - 黑色 / 深灰作为系统底色
   - 冷青作为电子界面光
   - 酸黄作为警告色
   - 橙红到红色作为情绪失控和危险升级信号
3. 背景、主面板、词云容器、图表主线、图表面积填充、右上角状态卡现在共享同一套主题变量。
4. 字体分层重做：
   - `Orbitron` 负责标题、数字、状态标签
   - `Rajdhani` 负责正文和数据界面文案
   - `Noto Sans SC` 负责中文回退和稳定显示
5. 词云和烟花粒子颜色改为从主题变量注入，不再写死一套固定配色。
6. 保留强烈的故障和告警感，但避免直接照搬具体作品画面。

## 验证

建议执行：

```bash
cargo test
```

生成报告页后确认：

1. 整页不再是局部强烈、整体平淡，而是形成统一的未来压迫感。
2. `SBAI` 越高，整页颜色越偏危险与失控。
3. 图表、词云、右上角卡片在色相上保持一致，不再互相脱节。
4. 标题、数字、标签的字体辨识度明显提升，不再像默认系统页面。
