# Bug Fixed: 词云显示过小且不可缩放

日期：2026-04-13

## 问题

报告页里的词云虽然已经渲染出来了，但有三个明显问题：

1. 词云实际显示尺寸偏小，看起来没有占满所在容器。
2. 在高分屏上字会进一步缩小，阅读性很差。
3. 用户无法放大、缩小或拖动查看密集区域。

## 根因

第一版词云使用的是 `wordcloud2.js` + `<canvas>`。

原实现把 `canvas.width` 和 `canvas.height` 乘上了 `devicePixelRatio`，但没有同步把 CSS 显示尺寸按同样比例调整，导致浏览器把整张画布缩回容器大小，视觉上就变成“内容更小了”。

同时，词云区域只有一次按容器大小的重绘，没有额外的 viewport/transform 层，所以也不存在缩放和平移能力。

## 修复

本次修复做了这些调整：

1. 去掉词云画布的错误 DPR 缩放逻辑，改为按可视容器实际尺寸绘制。
2. 最终改为无外部依赖的球形词云，替换平面词云实现。
3. 词云支持：
   - 拖拽旋转
   - 滚轮缩放
   - 自动慢速旋转
4. 给词云容器增加稳定的最小高度，避免在网格布局里显得过扁或过小。
5. 避免继续依赖外部 3D 库脚本，因为本报告是本地 `file://` 打开的单文件 HTML，外部依赖会增加加载失败面。
6. 进一步排查卡顿后，把词球改成“词图预渲染 + 每帧 drawImage”，不再每帧重复 `strokeText`/`fillText`。
7. 烟花背景重新增强了一版，但仍保留了较轻的粒子数量和自动停止，避免重新回到高负载。
8. 鼠标拖拽方向按相反方向处理，并去掉上下半圈限制，允许完整 360 度翻转。
9. 词球进一步按高频词裁剪，只保留前 32 个词，避免所有词都长期参与绘制。
10. 空闲状态下降低词球刷新频率，交互时再恢复完整帧率。
11. 背面的词进一步降低透明度，并对过小/过远词做简单裁剪。
12. 日趋势图改用 Apache ECharts 6，直接启用 `dataZoom`，支持滚轮缩放、拖拽平移和自适应铺满图表面板。
13. 补充了渲染结构测试，防止交互节点在后续修改中被删掉。

## 验证

建议执行：

```bash
cargo test
cargo run
```

打开生成的 HTML 后确认：

1. 词云默认已经占满大部分面板区域。
2. MacBook/Retina 屏幕上字体不会再异常缩小。
3. 鼠标拖拽可以旋转词球，滚轮可以缩放。
4. 页面直接以单文件 HTML 打开时，词球也能正常加载。
